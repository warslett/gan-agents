# Research Index

## Problem

> "What software architecture should I use for a virtual rummikub game allowing multiple players to play against each other or against an AI over the web?"

## Resources

1. [Rummikub - Game Rules and Mechanics](resource_1.md) — Complete breakdown of Rummikub's rules, tile sets, gameplay mechanics, scoring, and variations that define the requirements for any implementation.
2. [Colyseus - Multiplayer Game Framework](resource_2.md) — Overview of the Colyseus Node.js framework for authoritative game servers with real-time state synchronization, rooms, and matchmaking.
3. [Socket.IO - Real-Time Communication Library](resource_3.md) — Details on Socket.IO's bidirectional communication capabilities, rooms, events, and how it serves as a lower-level building block for multiplayer games.
4. [WebRTC Data Channels for Multiplayer Games](resource_4.md) — MDN documentation on using peer-to-peer WebRTC data channels for game communication, including reliable vs. unreliable channels and trade-offs.
5. [boardgame.io - Framework for Turn-Based Games](resource_5.md) — Overview of the boardgame.io framework purpose-built for turn-based board games, including its AI/bot support, secret state handling, and phase system.
6. [Rummikub AI and Solver Algorithms](resource_6.md) — Survey of algorithmic approaches for Rummikub AI including ILP solvers, MCTS, greedy heuristics, and the computational complexity of tile placement.
7. [Multiplayer Game Architecture Patterns](resource_7.md) — Comprehensive overview of architecture choices (client-server, P2P), communication protocols (WebSocket, REST, WebRTC), state sync strategies, and scaling patterns.
8. [Frontend Architecture for Web-Based Board Games](resource_8.md) — Comparison of frontend frameworks (React, Vue, Canvas/Phaser) and UI considerations specific to tile-based board games including drag-and-drop and responsive design.
9. [Game State Modeling for Rummikub](resource_9.md) — Detailed data modeling for Rummikub game state, visibility rules, turn flow state machine, validation logic, and persistence strategies.
